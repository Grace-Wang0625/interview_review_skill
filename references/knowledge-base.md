# 可选知识库接口

## 目录选择

按以下优先级确定目录：

1. 用户在当前请求中明确给出的路径。
2. 用户确认使用的历史路径。
3. Skill 根目录下的 `knowledge/`。

没有可用目录时，不报错终止；切换为联网检索。

## 支持与发现

先列出文件，不直接递归读取全部内容。优先读取与公司、岗位、项目或当前问题匹配的文件。常见资料包括 PDF、DOCX、Markdown、TXT、CSV、XLSX、网页导出和图片；调用相应文件能力可靠读取。

优先级通常为：

1. 用户简历、目标岗位 JD、公司材料。
2. 用户自己的项目资料、课程笔记和答案库。
3. 已整理的岗位题库、面经与评价标准。
4. 其他背景资料。

遇到大量文件时，先根据文件名和文本搜索缩小范围；只加载足以支持当前问题的片段。记录所用文件名，必要时标明页码或章节。

## 评价方式

- 先按候选人回答本身评价，再与知识库对照，避免把知识库措辞当唯一答案。
- 明确区分“知识库事实”“知识库建议”和“基于材料的推断”。
- 知识库与候选人回答冲突时，指出冲突及可能原因，不擅自裁定过时或错误的信息。
- 知识库中的面经仍是个体经验，不等同于公司官方标准。

## 联网回退

知识库没有覆盖时，联网搜索：`公司 + 岗位 + 轮次 + 面经/项目题/案例题`，并补充当前问题的专业标准查询。

- 优先近期且可追溯的来源。
- 公司、岗位与业务事实优先官方来源。
- 专业知识优先官方文档、教材、论文或公认权威资料。
- 面经至少交叉参考两个独立来源；只有单一来源时降低置信度。
- 不大段复制面经，不泄露付费或受限内容。
- 在复盘末尾列出实际采用的网络来源、链接、日期和用途。

若无法联网，清楚说明评价仅基于 PDF 与通用面试原则，不假装已查到外部标准。
